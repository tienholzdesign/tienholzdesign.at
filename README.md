# Tina CMS + Next.js Starter

A modern, headless CMS starter combining Tina CMS with Next.js, featuring a centralized theme configuration system.

## Quick Start

### Prerequisites

- Node.js 18+ with pnpm package manager
- Tina Cloud account (for CMS functionality)
- Vercel account (for production deployment)

### Installation

```bash
# Install dependencies
pnpm install

# Start development server
pnpm dev

# Open Tina CMS admin
# Visit http://localhost:3000/admin
```

## Theme Configuration System

This project uses a **centralized configuration system** to manage all theme, branding, and styling changes from a single file.

### Single Source of Truth: `config/theme-config.ts`

Instead of manually updating multiple configuration files across your project, all customization happens in one place:

```typescript
// config/theme-config.ts
export const themeConfig = {
  project: {
    domain: "yourdomain.com",
    applicationName: "Your Brand",
    authors: [{ name: "Your Name", url: "yourdomain.com" }],
  },
  colors: {
    primary: "#000000",
    secondary: "#767676",
    background: "#f5f5f5",
    // ... more colors
  },
  layout: {
    contentSize: "620px",
    wideSize: "1000px",
  },
  typography: {
    fontSize: {
      small: "1rem",
      medium: "1.125rem",
      large: "1.75rem",
      xLarge: "3rem",
    },
  },
  radixUI: {
    accentColor: "gray",
    appearance: "light",
    panelBackground: "translucent",
  },
  // ... more config
};
```

### Files Automatically Synced

When you update `config/theme-config.ts`, the following files are automatically populated:

| File                         | Values Used                         | How to Sync                       |
| ---------------------------- | ----------------------------------- | --------------------------------- |
| `styles/theme.css`           | Colors, layout, typography, spacing | Run `pnpm run generate-theme-css` |
| `app/layout.tsx`             | Radix UI theme configuration        | Automatically imported            |
| `content/config/config.json` | Application name, authors           | Run `pnpm run sync-config`        |
| `project.js` ⚡              | Production domain                   | Run `pnpm run sync-config`        |

**⚡ Auto-generated** — `project.js` is generated by the sync script. Don't edit it manually!

### Syncing Configuration

#### Automatic CSS Generation

Generate CSS variables from your configuration:

```bash
pnpm run generate-theme-css
```

This reads `config/theme-config.ts` and regenerates `styles/theme.css` with all your CSS custom properties.

#### Sync Generated Files

Some files (like `project.js` and `content/config/config.json`) are generated from the config. Update them with:

```bash
# Regenerate CSS, project.js, and config.json from theme-config.ts
pnpm run sync-all

# Or individually:
pnpm run generate-theme-css  # Generate CSS variables
pnpm run sync-config         # Generate project.js and config.json
```

The `sync-all` command automatically:

1. **Generates `styles/theme.css`** — CSS variables from color, layout, and typography config
2. **Generates `project.js`** — CommonJS file with your production domain
3. **Updates `content/config/config.json`** — Sets application name and authors

⚠️ **Important:** Don't edit these files manually — they'll be overwritten next time you run the sync scripts. Always update `config/theme-config.ts` instead.

### Configuration Structure

#### Brand & Project (`project.*`)

```typescript
project: {
  domain: "yourdomain.com",           // Production domain for SEO/sitemaps
  applicationName: "Your Brand",      // Shown in Tina CMS admin
  authors: [
    { name: "Company", url: "url" }   // Author metadata
  ]
}
```

#### Colors (`colors.*`)

```typescript
colors: {
  primary: "#000000",      // Main brand color
  secondary: "#767676",    // Secondary color
  foreground: "#000000",   // Text color
  background: "#f5f5f5",   // Background color
  tertiary: "#e6e6e6"      // Accent color
}
```

#### Layout (`layout.*`)

```typescript
layout: {
  contentSize: "620px",    // Default content width
  wideSize: "1000px"       // Wide content width
}
```

#### Typography (`typography.*`)

```typescript
typography: {
  fontSize: {
    small: "1rem",
    medium: "1.125rem",
    large: "1.75rem",
    xLarge: "3rem"
  },
  heading: {
    fontWeight: 300,
    letterSpacing: "-0.02em",
    lineHeight: 1.3
  }
}
```

#### Spacing (`spacing.*`)

```typescript
spacing: {
  unit: "1rem"; // Base spacing unit
}
```

#### Radix UI Theme (`radixUI.*`)

```typescript
radixUI: {
  accentColor: "gray",           // Primary accent color
  appearance: "light",           // "light" or "dark"
  panelBackground: "translucent" // Panel background style
}
```

## Adapting for a New Client/Theme

When customizing this project for a new client or WordPress theme:

### 1. Update Theme Configuration

Edit `config/theme-config.ts` with your brand values:

```typescript
export const themeConfig = {
  project: {
    domain: "clientdomain.com",
    applicationName: "Client Name",
    authors: [{ name: "Client Name", url: "clientdomain.com" }],
  },
  colors: {
    primary: "#YOUR_PRIMARY_COLOR",
    secondary: "#YOUR_SECONDARY_COLOR",
    // ... update all color values
  },
  // ... update other sections
};
```

### 2. Generate CSS Variables

```bash
pnpm run generate-theme-css
```

### 3. Sync Configuration Files

```bash
pnpm run sync-config
```

### 4. Update Fonts

If using custom fonts:

1. Copy font files to `public/fonts/`
2. Update `app/fonts.ts` with your font configuration
3. Rebuild: `pnpm build`

### 5. Update Favicons

Replace favicon files to match your brand:

```bash
# Update favicon
cp your-icon.ico public/favicon.ico

# Update app icon (PNG for modern browsers)
cp your-icon.png app/icon.png
```

### 6. Set Up Tina CMS

1. Create account at [Tina Cloud](https://dashboard.tinajs.io/)
2. Create new project for your client
3. Generate Client ID and Token
4. Add to `.env.local`:
   ```bash
   NEXT_PUBLIC_TINA_CLIENT_ID=your_client_id
   TINA_TOKEN=your_token
   ```

### 7. Deploy to Vercel

1. Connect GitHub repository to [Vercel](https://vercel.com)
2. Add environment variables to Vercel
3. Deploy

For more detailed customization steps, see [THEME_ADAPTATION_GUIDE.md](THEME_ADAPTATION_GUIDE.md).

## Environment Variables

Create a `.env.local` file in the project root:

```bash
# Tina CMS Authentication
NEXT_PUBLIC_TINA_CLIENT_ID=your_client_id_here
TINA_TOKEN=your_private_token_here
```

⚠️ **Never commit `.env.local` to version control.**

### Generated Files

The following files are auto-generated and should not be committed:

- `project.js` — Generated by `pnpm run sync-config` from `config/theme-config.ts`

These are already in `.gitignore`. Always edit the source configuration instead of these generated files.

## Troubleshooting

### CSS Variables Not Applying

1. Ensure `styles/theme.css` is imported in `styles/main.css`
2. Verify CSS variables are generated: `pnpm run generate-theme-css`
3. Clear build cache: `rm -rf .next`
4. Restart dev server

### Configuration Changes Not Reflecting

1. **For CSS**: Run `pnpm run generate-theme-css`
2. **For JSON/JS**: Run `pnpm run sync-config`
3. **For Radix UI theme**: Changes in `app/layout.tsx` auto-import, just restart dev server

### Font Files Not Loading

1. Verify files are in `public/fonts/`
2. Check file paths in `app/fonts.ts`
3. Ensure TTF/OTF format is correct
4. Clear cache: `rm -rf .next`

### Lint Errors

Run type checking and linting:

```bash
pnpm lint
```

Fix issues automatically where possible with ESLint's `--fix` flag (included in `pnpm lint`).
